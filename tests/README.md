# 机场货站提货预约系统 - 测试说明

## 测试场景概览

本目录包含系统的自动化测试脚本，用于验证系统的核心业务流程和异常处理机制。

## 测试场景列表

### 场景1：初始化测试数据
- 初始化运单、车辆等基础测试数据
- 验证测试环境准备

### 场景2：成功提货完整流程
验证正常的提货流程：
1. 货代提交预约（运单、车辆、司机、预计到场窗口）
2. 仓库货权确认（核验提货单、欠费、监管锁定）
3. 加入月台排队
4. 安保检查放行（证件、排队顺序、窗口过期）
5. 开始提货
6. 完成提货
7. **强校验验证**：完成提货后不能撤回

### 场景3：货权审核失败场景
验证货权审核不通过的异常处理：
- 提货单与运单信息不符
- 货物欠费
- 货物被监管锁定
- 系统记录驳回原因

### 场景4：安保驳回场景
验证安保检查不通过的异常处理：
- 车辆证件过期
- 排队顺序错误
- 预约窗口过期
- 系统记录驳回原因并离开队列

### 场景5：车辆变更场景
验证货代变更车辆的处理：
- 货代变更车牌号、司机信息
- 记录变更原因
- **自动重新计算队列位置**

### 场景6：部分放货场景
验证仓库部分放货的处理：
- 仓库部分放货（如部分货物未到）
- 记录部分放货原因
- **自动重新计算队列位置
- 剩余货物可再次提货

### 场景7：货权未确认不能排队
**强校验验证：
- 未进行货权确认的预约不能加入排队
- 后端强制拦截，返回错误

## 测试脚本使用方法

### 方法1：使用 Shell 脚本
```bash
cd tests
chmod +x test_scenarios.sh
./test_scenarios.sh
```

### 方法2：使用 Python 脚本（推荐，更详细）
```bash
cd tests
pip install requests
python test_api.py
```

## 测试前准备

1. 启动后端服务
```bash
cd ../backend
mvn spring-boot:run
```

2. 或使用 Docker 启动完整环境
```bash
docker-compose up -d
```

3. 确保服务健康检查
```bash
curl http://localhost:8080/actuator/health
```

## 核心验证点

### 状态流转健壮性验证：

- ✅ 完成提货后不能撤回
- ✅ 货权未确认不能排队
- ✅ 证件过期不能入场
- ✅ 车辆变更重新计算队列
- ✅ 部分放货重新计算队列
- ✅ 安保驳离开队列

### 操作审计验证：
- ✅ 每次操作记录操作人
- ✅ 状态变更记录前后快照
- ✅ 异常原因可追溯

### 并发控制验证：
- ✅ 乐观锁防止并发修改
- ✅ 分布式锁保护队列操作
